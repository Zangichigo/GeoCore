# Getting Started

Welcome to GeoCore.

This guide explains how to build the project, run the tests and understand the project structure.

---

# What is GeoCore?

GeoCore is a modern C++ geospatial engine.

It provides reusable geospatial objects and algorithms while remaining independent from hardware, business domains and external services.

GeoCore is designed to be simple, lightweight and easy to understand.

---

# Project Structure

```
GeoCore/

Engine/
Demo/
Tests/
Documentation/
.github/
```

- **Engine** contains the library source code.
- **Demo** contains demonstration programs.
- **Tests** contains the unit tests.
- **Documentation** contains the project documentation.

---

# Building GeoCore

Configure the project:

```bash
cmake -B build
```

Build the project:

```bash
cmake --build build
```

---

# Running the Demo

Execute the demo application after building.

The demo showcases the current capabilities of GeoCore.

---

# Running the Tests

Run all unit tests:

```bash
ctest --test-dir build
```

Every test should pass before submitting changes.

---

# Continuous Integration

Every pull request is automatically validated using GitHub Actions.

Contributors should ensure that:

- The project builds successfully.
- All tests pass.
- No warnings are introduced when reasonably avoidable.

# Where to Start?

If you are new to the project, we recommend the following order:

1. Read the Manifesto.
2. Read the Design Principles.
3. Explore the Demo application.
4. Run the unit tests.
5. Browse the Engine source code.

This provides a complete overview of GeoCore in less than one hour.
