# GeoCore Roadmap

This roadmap describes the long-term direction of GeoCore.

GeoCore is intentionally designed to remain lightweight, modern and easy to understand.

The objective is not to become the largest geospatial library, but one of the most pleasant to use.

---

# Vision

GeoCore is a modern C++ geospatial engine.

It provides reusable geospatial objects and algorithms that can be used in any application.

GeoCore remains independent from:

- Hardware
- Business domains
- External services
- Data formats

External formats are provided as adapters built on top of the engine.

# Out of Scope

GeoCore intentionally does **not** include:

- Smart-home integrations
- Home Assistant support
- MQTT
- Zigbee
- LoRa
- GPS hardware drivers
- Mapping applications
- Cloud services
- User interfaces

These belong to applications built on top of GeoCore.

---

# Roadmap Philosophy

GeoCore evolves conservatively.

A capability is accepted only if it answers **yes** to all three questions:

1. Does it solve a common geospatial problem?
2. Does it keep the API simple and intuitive?
3. Does it belong inside the engine rather than an adapter or an application?

If one answer is "no", the capability probably does not belong in GeoCore.

---

# Long-Term Objective

GeoCore aims to become a trusted foundation for geospatial software.

Applications should be able to build upon GeoCore without GeoCore ever becoming coupled to them.

Examples include:

- OpenFence
- GIS software
- Robotics
- Drone software
- Scientific simulations
- Asset tracking
- Outdoor navigation
- Research projects
