# Capability Template

This template defines the standard workflow for introducing a new capability into GeoCore.

Every capability should follow the same structure to ensure consistency across the project.

---

# Capability ID

Example:

GC-0035

---

# Name

Example:

Rectangle

---

# Objective

Describe what problem this capability solves.

Keep the description concise.

---

# Module

Specify the GeoCore module.

Examples:

- Core
- Math
- Geometry
- Movement
- Tracking
- Algorithms
- Formats

---

# Public API

Describe the public interface.

Example:

```cpp
Rectangle rectangle(
    Position southWest,
    Position northEast);
```

---

# Responsibilities

Clearly describe what the capability does.

Also describe what it intentionally does not do.

---

# Dependencies

List the required GeoCore components.

Example:

Depends on:

- Position

Does not depend on:

- Track
- Movement
- Formats

---

# Unit Tests

List every expected test.

Example:

- Rectangle contains point
- Rectangle outside point
- Rectangle contains corners
- Rectangle accessors

---

# Demonstration

Describe what should be added to the Demo application.

Example:

Rectangle test

Inside : true

Outside : false

---

# Documentation

Documentation to update.

Examples:

- README
- Roadmap
- API Reference
- Examples

---

# Future Improvements

List possible future extensions.

These should not be implemented unless they become necessary.

Example:

- intersects()
- width()
- height()
- center()

---

# Review Checklist

Before merging the capability, verify:

- [ ] Architecture validated
- [ ] Public API reviewed
- [ ] Compiles successfully
- [ ] Unit tests added
- [ ] All tests pass
- [ ] Demo updated
- [ ] Documentation updated
- [ ] GitHub Actions successful

---

Additional design decisions or ADR references.
